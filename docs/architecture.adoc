== Overview

This project provisions isolated Ubuntu VMs with Multipass for agentic
development work. The architecture keeps the VM boundary as the primary
security control, but replaces the old Bash launch workflow with a thin Go
`create` CLI that reads `devvm.yaml`, launches or clones a VM, transfers
scoped provisioning files into the guest, and runs post-boot setup through
Multipass-native commands.

== Security Architecture

=== Isolation Boundary

The security model depends on the Multipass VM boundary, not on in-guest MAC
policies or host directory mounts. Repositories live inside the guest
workspace, and host credential directories are not mounted into the VM.

=== Provisioning Boundary

Secrets and setup inputs are delivered as scoped files copied into the guest
during startup. The host environment is not inherited into the VM. Only the
files required for the current session are staged and transferred.

=== Access Model

The VM is entered through `multipass shell`. SSH is disabled. The `opencode`
user is the runtime identity for agent tooling, and bootstrap removes sudo
access before the environment is handed to the developer.

=== Network Egress Controls

UFW enforces default-deny inbound and outbound policies. Outbound access is
limited to the ports configured in `devvm.yaml`, typically:

* 443/tcp for HTTPS APIs and package downloads
* 53 for DNS resolution

Port 22/tcp can be added when SSH-based git access is required.
