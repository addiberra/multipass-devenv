== Overview

This project is a minimal Go CLI for launching an isolated Ubuntu VM with
Multipass from a checked-in machine config and referenced cloud-init file. The
repository ships both reusable template artifacts under `templates/` and a
repo-specific root setup for self-hosting. The VM boundary remains the primary
security control. The host-side tool validates the machine config, launches the
VM, waits for cloud-init to finish, and may then attach one explicit repo-
scoped host mount.

== Security Architecture

=== Isolation Boundary

The primary security boundary is the Multipass VM. The host CLI may mount one
explicitly configured project directory into the guest after launch, but does
not expose broader host paths.

=== Provisioning Boundary

Provisioning is defined by the cloud-init file referenced from `devvm.yaml`
plus an optional post-boot repo mount declared in that same config. The root
files are this repository's self-hosting setup, while `templates/` contains the
copy-and-adapt starter pair for other repos. The CLI does not inject secrets or
perform general file transfer.

=== Access Model

The VM is accessed through Multipass-managed guest operations such as
`multipass shell` and `multipass exec`. The runtime user inside the VM is
non-root and is not granted sudo access.

=== Network Egress Controls

UFW is configured inside the guest by cloud-init with default-deny inbound and
outbound policies, plus explicit outbound allows defined in the checked-in
cloud-init file.
