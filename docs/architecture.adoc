== Overview

This project is a minimal Go CLI for launching an isolated Ubuntu VM with
Multipass from two checked-in inputs: `devvm.yaml` and `opencode-sandbox.yaml`.
The VM boundary remains the primary security control. The host-side tool does
not render extra provisioning data, transfer files after boot, or manage
repository cloning. It validates the machine config, launches the VM, and waits
for cloud-init to finish.

== Security Architecture

=== Isolation Boundary

The primary security boundary is the Multipass VM. The host CLI does not mount
host directories into the guest.

=== Provisioning Boundary

Provisioning is entirely defined by the checked-in `opencode-sandbox.yaml`
cloud-init file. The host-side CLI does not inject secrets or perform post-boot
file transfer.

=== Access Model

The VM is accessed through `multipass shell`. SSH is disabled in the guest. The
runtime user inside the VM is non-root and is not granted sudo access.

=== Network Egress Controls

UFW is configured inside the guest by cloud-init with default-deny inbound and
outbound policies, plus explicit outbound allows defined in the checked-in
cloud-init file.
